# setup_snicar module

This module sets up BioSNICAR by creating instances of all the relevant classes using the data provided in `inputs.yaml`.

**Note:** For most use cases, the higher-level `run_model()` function (in `biosnicar.drivers.run_model`) is recommended instead of calling `setup_snicar()` directly. `run_model()` handles the full pipeline (setup, optical properties, impurity mixing, radiative transfer) in a single call and accepts keyword overrides for any parameter. See the [User Guides](../user-guides/simple-albedo-prediction) for examples.

## setup_snicar

Calls internal functions and returns class instances as expected by BioSNICAR model.

### Args
- `input_file`: path to a YAML configuration file, or `"default"` to use the built-in `inputs.yaml`

### Returns
- `ice`: instance of Ice class
- `illumination`: instance of Illumination class
- `rt_config`: instance of RTConfig class
- `model_config`: instance of ModelConfig class
- `plot_config`: instance of PlotConfig class
- `impurities`: array of instances of Impurity class


## build_classes

Instantiates classes according to config in yaml file.

### Args
- `input_file`: path to a YAML configuration file

### Returns
- `ice`: instance of Ice class
- `illumination`: instance of Illumination class
- `rt_config`: instance of RTConfig class
- `model_config`: instance of ModelConfig class
- `plot_config`: instance of PlotConfig class

## build_impurities_array

Creates an array of instances of Impurity with properties defined in yaml file.

### Args
- `input_file`: path to a YAML configuration file

### Returns
- `impurities`: array of instances of Impurity
