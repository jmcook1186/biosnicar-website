# run_model module

This module provides the single entry point for running the BioSNICAR forward model. It handles the full pipeline (setup, optical properties, impurity mixing, radiative transfer) in a single call and accepts keyword overrides for any model parameter.

## run_model

Run the BioSNICAR forward model and return outputs.

Builds model objects from the input file, applies any keyword overrides to ice/illumination/impurity parameters, then runs the full pipeline.

### Args
- `input_file`: path to a YAML configuration file, or `"default"` to use the built-in `inputs.yaml`
- `solver`: `"adding-doubling"` (default) or `"toon"`
- `validate`: if `True`, run input validation before the forward model
- `plot`: if `True`, display a spectral albedo plot after the run
- `**overrides`: parameter overrides applied before running the model (see supported keys below)

### Returns
- `outputs`: instance of `Outputs` class with albedo, BBA, absorbed fluxes, etc.

### Raises
- `ValueError` if `solver` is not recognised or an override key is unknown

### Supported override keys

| Key | Applies to | Behaviour |
|-----|-----------|-----------|
| `solzen` | `illumination.solzen` | Scalar, triggers `calculate_irradiance()` |
| `direct` | `illumination.direct` | Scalar, triggers `calculate_irradiance()` |
| `incoming` | `illumination.incoming` | Scalar, triggers `calculate_irradiance()` |
| `rds` | `ice.rds` | Broadcast to all layers if scalar |
| `rho` | `ice.rho` | Broadcast to all layers if scalar |
| `dz` | `ice.dz` | Broadcast to all layers if scalar, or pass a list |
| `lwc` | `ice.lwc` | Broadcast to all layers if scalar |
| `layer_type` | `ice.layer_type` | Broadcast to all layers if scalar |
| `shp` | `ice.shp` | Grain shape; broadcast to all layers if scalar |
| `grain_ar` | `ice.grain_ar` | Grain aspect ratio; broadcast to all layers if scalar |
| `cdom` | `ice.cdom` | CDOM concentration; broadcast to all layers if scalar |
| `water` | `ice.water` | Liquid water coating radius; broadcast to all layers if scalar |
| `hex_side` | `ice.hex_side` | Hexagonal column side length; broadcast to all layers if scalar |
| `hex_length` | `ice.hex_length` | Hexagonal column length; broadcast to all layers if scalar |
| `shp_fctr` | `ice.shp_fctr` | Shape factor; broadcast to all layers if scalar |
| `impurity_{i}_conc` | `impurities[i].conc` | Scalar: applied to first layer only (rest zeroed); list: applied directly |

### Layer count changes and automatic resizing

When you pass a list-valued ice override (e.g. `dz`, `rds`, `rho`) with a different length than the default configuration, `run_model` automatically resizes **all** per-layer attributes to match the new layer count. This includes both ice properties (`cdom`, `shp`, `water`, `hex_side`, `hex_length`, `shp_fctr`, `grain_ar`, `lwc`) and **impurity concentrations**.

- **Ice attributes** (e.g. `dz`, `rds`, `rho`, `grain_ar`) that aren't explicitly overridden are extended by **repeating their last value**, or truncated to the new length. This ensures physical properties like grain radius and density are never zero-filled.
- **Impurity concentrations** that aren't explicitly overridden are **zero-padded** to the new length, or truncated. Zero is the correct default since additional layers should be assumed clean unless specified otherwise.

For example, the default YAML config has 2 layers. If you pass `dz=[0.02, 0.05, 0.05, 0.05, 0.83]` (5 layers), all ice attributes and impurity concentrations are resized to length 5 before your overrides are applied. Without this resizing, `mix_in_impurities` would fail with a broadcast shape mismatch between the 5-layer ice arrays and the 2-element impurity arrays.

```py
# 5-layer model — only need to specify the overrides you care about.
# Non-overridden impurity concentrations are automatically zero-padded to 5 layers.
outputs = run_model(
    dz=[0.02, 0.05, 0.05, 0.05, 0.83],
    rds=[800, 900, 1000, 1100, 1200],
    rho=[500, 600, 700, 750, 800],
    impurity_1_conc=[40000, 10000, 0, 0, 0],  # snow algae in top 2 layers
    # impurity_0_conc and impurity_2_conc are auto-resized to [0, 0, 0, 0, 0]
)
```

### Examples

```py
from biosnicar.drivers.run_model import run_model

# Run with all defaults
outputs = run_model()
print(outputs.BBA)

# Override specific parameters
outputs = run_model(solzen=50, rds=1000)
print(outputs.albedo)

# With validation and plotting
outputs = run_model(solver="adding-doubling", validate=True, plot=True)

# Set impurity concentration (black carbon in first layer)
outputs = run_model(impurity_0_conc=500)

# Use as top-level convenience
import biosnicar
outputs = biosnicar.run_model(solzen=60, rds=200)
```
